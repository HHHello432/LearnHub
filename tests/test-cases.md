## 测试用例标准结构

| 字段 | 说明 | 示例 |
|------|------|------|
| **用例编号** | 唯一标识，格式：`模块缩写-序号` | `AUTH-001` |
| **测试模块** | 所属功能模块 | 用户登录 |
| **用例标题** | 简明扼要描述测试目的 | 正确用户名密码登录成功 |
| **前置条件** | 执行前需准备好的环境、数据或状态 | 已有注册账号 admin/123456 |
| **测试步骤** | 详细、可逐条执行的操作步骤 | 1.打开登录页 2.输入正确账号密码 3.点击登录 |
| **测试数据** | 该用例使用的具体输入值 | username=admin, password=123456 |
| **预期结果** | 期望系统表现，必须明确可验证 | 登录成功，跳转首页，侧边栏显示用户名 |
| **实际结果** | 执行后填写 | （留空） |
| **测试状态** | 通过 / 失败 / 阻塞 | — |
