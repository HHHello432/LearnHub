## 1. 项目背景与目标

### 1.1 背景

在学习过程中，学习者通常面临以下问题：
- **缺乏专注力管理工具**：无法有效跟踪学习时间
- **任务管理混乱**：无法区分轻重缓急
- **习惯难以坚持**：缺少视觉反馈和激励机制
- **学习数据分散**：笔记、日记、打卡等数据割裂在不同平台

### 1.2 项目目标

构建一个**一体化个人学习平台**，实现从**学习 → 记录 → 反馈 → 激励**的完整闭环，帮助用户：
1. 提升学习专注力（番茄钟）
2. 高效管理任务（四象限法）
3. 养成良好习惯（打卡追踪）
4. 记录学习过程（笔记/日记）
5. 获得正向反馈（积分/等级/商城）

### 1.3 目标用户

| 用户类型 | 说明 | 权限范围 |
|----------|------|---------|
| 普通用户 | 注册用户 | 使用所有学习功能、积分兑换 |
| 访客 | 无需注册的浏览用户 | 浏览演示数据、部分功能受限 |
| 管理员 | 平台管理者 | 用户管理、商品管理、系统配置 |

## 3. 非功能需求

### 3.1 性能指标

| 指标 | 目标值 |
|------|--------|
| API 响应时间（P95） | < 300ms |
| 并发用户支持 | 50+ 虚拟用户 |
| 页面首次加载 | < 2s |
| 页面切换 SPA | < 200ms |

### 3.2 安全需求

| 需求 | 方案 |
|------|------|
| 密码安全 | bcryptjs 加密存储 |
| 认证 | JWT Bearer Token，7天过期 |
| 角色控制 | 路由守卫 + API 中间件双重校验 |
| 写操作保护 | 访客禁止写入（rejectGuest 中间件） |
| XSS 防护 | 输入过滤、输出转义 |

### 3.3 可用性需求

| 需求 | 方案 |
|------|------|
| 100% 正常运行 | Docker 自愈重启 |
| 数据持久化 | PostgreSQL 持久卷 |
| 健康检查 | Docker healthcheck |
| 错误处理 | 统一错误中间件 + 前端 Toast 提示 |

## 4. 数据流

```
用户操作 → Vue 组件 → API 封装模块 → Axios → Express 路由 → Prisma ORM → PostgreSQL
                                                                         ↓
         Vue 组件 ← 数据渲染 ← JSON 响应 ← 统一错误处理 ← 路由中间件 ←────┘
```

### 核心积分闭环

```
打卡 +5分 ──┐
任务 +20分 ──┤
笔记 +3分 ──┼──→ 积分增长 → 等级提升 → 商城兑换 → 积分减少
番茄 +10分 ─┘
              ↑
        取消操作自动扣回
```
